# 贡献指南

## 代码组织

- 新业务功能放到：`src/main/java/indi/midreamsheep/vegetable/backend/features/<功能名>`。
- 通用横切能力放到：`.../common`（避免把具体业务规则写进 `common`）。
- 技术适配与配置放到：`.../infrastructure`（如 MyBatis-Plus、Redis、JWT、OpenAPI 等）。

## 注释规范（强制）

- 所有代码必须具备文档级/方法级/常量级注释：见 `docs/代码注释规范.md`。

## 日志规范（强制）

- 日志级别/格式/敏感信息/traceId：见 `docs/日志规范.md`。

## API 规则

- 公共 HTTP API 统一使用 `/api/v1` 前缀。
- Controller 统一返回 `ApiResponse`。
- 新增错误码必须进入 `common/error/ErrorCode`，并保持“稳定整数”不随意变更。

## 测试

- 新增接口时，优先补充对应的 Web 层测试（MockMvc）。
- 单元测试尽量贴近对应 feature，避免跨模块耦合。
