# 代码注释规范（强制）

本规范为 **强制执行**：所有 Java 代码必须具备 **文档级（类/接口/枚举/记录）**、**方法级（含构造方法）**、**常量级（含枚举常量与 `static final` 常量）** 的标准注释。

项目通过 Checkstyle 在构建阶段强制校验：`./mvnw validate` / `./mvnw test` 会在发现缺失注释时直接失败。

## 1. 文档级注释（必需）

适用对象：`class` / `interface` / `enum` / `record`。

要求：

- 采用 Javadoc 形式：`/** ... */`
- 第一行描述该类型的职责（中文）

推荐模板：

```java
/**
 * （一句话）说明该类型的职责与边界。
 */
public class Xxx {
}
```

## 2. 方法级注释（必需）

适用对象：所有方法与构造方法（包括 `private`、`@Override` 方法、Spring Bean 方法等）。

要求：

- 采用 Javadoc 形式：`/** ... */`
- 至少描述该方法做什么/输入输出含义/关键副作用（中文）
- 有参数时建议补充 `@param`；有返回值时建议补充 `@return`；可能抛异常时建议补充 `@throws`

推荐模板：

```java
/**
 * （一句话）说明方法目的与行为。
 *
 * @param arg 参数说明
 * @return 返回值说明
 */
public Xxx doSomething(String arg) {
    return ...;
}
```

## 3. 常量级注释（必需）

适用对象：

- 枚举常量（`enum` 的每个枚举值）
- `static final` 常量（无论访问级别）

命名约定（强烈建议）：

- 常量命名使用“大写下划线”（例如 `SECURITY_SCHEME_NAME`）。
- 项目中对“常量级注释”的自动校验会基于该命名约定识别常量；不遵守命名约定可能导致校验无法覆盖。

要求：

- 采用 Javadoc 形式：`/** ... */`
- 描述常量含义与使用场景（中文）

推荐模板：

```java
/**
 * JWT 在 OpenAPI 中的安全方案名称。
 */
public static final String SECURITY_SCHEME_NAME = "BearerAuth";
```

## 4. 不推荐的写法

- 空注释或无意义注释（例如“TODO”“test”“xxx”）
- 仅英文注释（除非是标准术语/字段名/协议名）
- 只在调用处解释，不在定义处解释（定义处必须有注释）
